# FHIR-51307: Provide Guidance On Historical Data

## Description

The eHN guidelines and this implementation guide do not specify how historical data, which may not conform to these profiles, should be handled. We would recommend that a date be set beyond which new imaging data SHOULD conform to this IG, but that record holders MAY surface other data that they believe is clinically relevant, even if it does not conform to the profiles. This promotes patient safety by ensuring that data is available, while reducing the need for healthcare organizations to convert / manually remediate legacy data.

This is also in keeping with FHIR’s broader guidance that servers may return results that go beyond what a client specified in their request, if they deem it relevant.

Where we will also explain that it is not up to this specification to indicate what approach is required but that newer systems SHOULD implement the structured version of the FHIR report and that we expect that regulators may require this from a certain point in time.
