# FHIR-51355: Use-case description is severly lacking

## Description

Use-case description is severly lacking, this part of the IG is a must have to be able to give meaningfull comments about the implementation guide. Without proper information about what use-cases the implementation should support including what Actors are involved and their roles obligations and transactions involved, the IG does not make much sense to publish or comment.

This part is slightly better described in related IG's that are also part of the XtEHR specification, but the example linked below is still missing a clear definition of transactions.
