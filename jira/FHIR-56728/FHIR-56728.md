# FHIR-56728: Extensible SNOMED binding on Procedure.code requires extensive mapping; allow LOINC

## Description

An extensible binding on SNOMED codes for Procedure.code is challenging and requires extensive mapping effort.

Extensible means that if a valid value exists in the list it must be used. Many organizations map to LOINC codes rather than SNOMED codes for procedures to have greater specificity. See the LOINC/RSNA Radiology Playbook.

While code.coding is 0..*, such that a Procedure could be mapped to both LOINC and SNOMED (if they are compatible) that adds complexity.

Proposed resolution:

Research whether there is a clear mapping from LOINC to SNOMED, and if not, consider allowing LOINC.
