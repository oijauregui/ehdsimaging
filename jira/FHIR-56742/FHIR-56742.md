# FHIR-56742: Not all RIS support discrete history capture; making patientHistory MS forces a workflow change. Remove MS.

## Description

Not all RIS / result entry systems support radiologists indicating additional history discretely, and in many healthcare organizations this will not be part of the radiologist's workflow. Therefore, this would add extra burden to the clinician simply to meet the requirements of this specification.

Proposed resolution:

Extension:patientHistory should not be MS.

If structured patient history is wanted, the Patient Summary IG already defines a model for it; the imaging IG should reference Patient Summary's history representation and allow a Composition to combine the two, rather than introduce a generic-vehicle history slot on the imaging report itself.

Reporter(s): Brian Swinkels, Epic
