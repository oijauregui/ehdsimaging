# FHIR-56727: Profile requires dicomSrInstance 1..1 and derivedFrom 1..*; relax both to 0..*

## Description

.derivedFrom should not be 1..* with dicomSrInstance set to 1..1.

Radiation dose information may be available that wasn't communicated via dicomSR instances. For example, technologists / radiographers may directly document radiation exposure. DICOM MPPS was also used to file radiation dose information until very recently.

Proposed resolution:

Make dicomSrInstance 0.._ and derivedFrom 0.._.
