# FHIR-51172: revise dependency from ImIheMhdDocumentReference

## Description

ImManifestIheMhdDocumentReference declares a dependency from a ImIheMhdDocumentReference that is not a “real” MHD document reference profiles this could be misleading. 


Check what are the actual needs and based on this verify if one of the several DocumentReference MHD profiles fits for the purpose.

In this case define a concrete dependency from the IHE profile;

If not, remove any reference to IHE MHD otherwise it could be misleading.

For example IHE MHD DocRef Minimal has DocumentReference.content.attachment.data 0..0 is this what we want ?

## Resolution

- Status: Applied
- Resolution: Persuasive with Modification

We will remove the profile from the R5 release, and add a reference to the IHE-MHD profile in the R4 release.

The DocumentReference profile will remain. A note will be added to the description that the intend of this resource is to be used in a MHD context but whether such context will have a R5 representation is still not decided.
