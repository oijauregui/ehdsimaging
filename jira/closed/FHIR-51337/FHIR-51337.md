# FHIR-51337: Accession Numbers Should Not Be Regarded as Globally Unique (They aren't in exception cases)

## Description

This is a callout for a point we spotted in Release 1.1 of the eHN Guidelines that should be corrected. We don’t believe that we saw this reflected in the Implementation Guide, but are

Page 12 of the text (under Article 2 – Terms and Conditions) references that Accession Numbers are globally unique:



Accession Numbers are not typically prefixed with any registration authority today. Even within an EHR, an accession number need not be unique, especially if data has been converted from legacy systems or if there are multiple imaging servers present. This may make it challenging to prevent duplicates, even with the Data Type Profile on Imaging Accession Number Identifier.

EHRs and administrators of EHRs typically work to ensure that accession numbers are unique for a given Patient and Imaging Server to ensure that links to imaging (such as via a Web PACS) function.

## Resolution

- Status: Applied
- Resolution: Persuasive with Modification

*Add the following documentation related to accession numbers:*

_Within a Hospital Information System environment, accession numbers will *usually* uniquely identify a manifest or report but are not required to *always* map 1:1 with a report or manifest due to certain imaging workflows. The imaging report may not be uniquely identified by accession number in an overread scenario (two radiologists reading a single imaging study). The manifest may not be uniquely identified by accession number in the case that the RIS imaging request (accession number assigner) requires two imaging modalities to perform the exam, and thus produces two image manifests._

_In order to ensure uniqueness *across* Hospital Information Systems, accession numbers are required to have a system value that identifies the namespace for the issuing authority (ensuring their global uniqueness)._
