# FHIR-51301: ConceptMap representation is hard to review

## Description

The representation/rendering of the mapping to the XtEHR model makes it hard to review. Please revise it to show the full path of the source and target resource.

(Source: Rutt Lindstrom )

## Resolution

- Status: Applied
- Resolution: Persuasive

We will improve the representation of the mapping when the specification is updated to the new XtEHR model.

This will follow the approach taken in MPD ([Mapping to Profiles - HL7 Europe Medication Prescription and Dispense v0.1.0-ballot|https://hl7.eu/fhir/mpd-r5/profiles.html]). The mapping will present the full path. 

Instead of ConceptMaps, a set of markdown tables will be included. This will also greatly reduce the number of artifacts in the specification and the reduce the number of build errors.

In later versions of the specification, the ConceptMaps might be re-introduced.
