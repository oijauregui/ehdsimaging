# FHIR-51171: Composition: remove fixed code for type

## Description

1)

Please remove the hardcoded value for type (e.g., imaging-report-v0-0-1), as it is not adopted by the current MyHealth@EU specifications.

This locally defined code functions similarly to a formatCode, effectively merging both the document type and format. However, this kind of information is intended to serve as document metadata and is not necessarily meant to be embedded within the document instance itself.

Furthermore, there are several practical reasons for not storing the specification (and its version) directly in the instance—similar to the rationale for avoiding use of meta.profile in instances. Each new publication would render previous instances non-conformant. If the same strategy were applied across derived guides, it could lead to an unwieldy proliferation of type codes.

I recommend continuing to use the IHE formatCode (also present in DocumentReference) to fulfill this role.

2)

remove the binding to the HL7 EU Imaging Report Types (extensible)
