# FHIR-51387: Add Design Consideration: Principle of Separating Imaging Manifest (PACS) from Imaging Report

## Description

Design Considerations should include the principle of separation of Image Data from Report data (e.g. PACS data vs RIS, but probably genericized to report/manifest provider).

This has come up several times in calls and tickets and is easy to misinterpret. Including info on this would also add clarity to the implementing systems trying to figure out which part of the spec they should care about.

Suggest new section in Design Considerations, covering some formalization of this decision in Confluence: Report and manifest - HL7 Europe - Confluence
