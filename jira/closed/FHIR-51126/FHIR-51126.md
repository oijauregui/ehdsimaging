# FHIR-51126: The IG structure / template should follow the general agreed one

## Description

The IG structure / template should follow the general agreed one: 

	Home: single page
	Introduction: multiple pages
	Functional
	Implementation
	About
	Artefacts
	&#91;link to Rx&#93;



Some of the existing pages - or part of them - should be moved consequently…

For example dependency should go in a separate page under About.

## Resolution

- Status: Applied
- Resolution: Persuasive

The new IG navigation structure will be as follows: 
 * Home -> stays with no children
 * Introduction
 ** Changes
 ** Dependencies
 ** References
 ** Adapting Guidelines for Country use
 * Functional
 ** System actors
 ** Use cases
 ** Transactions
 ** XtEHR model
 * Implementation
 ** XtEHR mapping
 ** Design considerations
 * About
 ** Authors and contributors
 ** Dependencies
 * Artifacts
 ** All
 ** Structural
 ** Imaging Report

Link to R4/R5
        
(REMOVED/MOVED ELEMENTS)    :    
 * Actors and transactions  (removed)
 ** System actors (moved  to Functional)
 ** Use cases (moved  to Functional)
 ** Transactions (moved  to Functional)
 * XtEHR model (removed)
 ** XtEHR model  (moved  to Functional)
 ** XtEHR mapping  (moved  to Implementation)
 * Imaging Manifest (migrated to the Manifest IG)
 * ImagingReport (moved to Artifacts)
 * Artifact.Mapping (removed, duplicated with XtEHR mapping)
 * Design considerations (moved to Implementation)
