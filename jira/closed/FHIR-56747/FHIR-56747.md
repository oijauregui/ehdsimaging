# FHIR-56747: Procedure, ServiceRequest, and Provenance entries on Resource Access Provider/Consumer require search parameters as SHALL even though no search interaction is required

## Description

Entries for Procedure, ServiceRequest, and Provenance do not require a search interaction, but nonetheless require search parameters. As the search parameters are given as SHALL, it is not clear why this is included. Below in the per-resource section they use SHALL language for the search parameters without any mention of the search as SHALL / SHOULD / MAY etc.

This applies to both Provider and Consumer capability statements.

Proposed resolution:

If search is not required no search parameters should be given, or if search is optional, it should be listed as either SHOULD or MAY, and the parameters marked as such.
