# FHIR-51335: DiagnosticReport.Category.Code Mixes Service Category and Report Category, Is too Strict

## Description

Imaging Diagnostic Report - HL7 Europe Imaging Study Report v0.1.0-ballot

Diagnosticreport.Category.Code has a reqiured slice for a LOINC code of 18748-4 that corresponds to Diagnostic Imaging Study Document.



	FHIR Core (Valueset-diagnostic-service-sections - FHIR v5.0.0) binds Category.code to a broad service buckets (imaging, cardiology, etc). This drives DiagnosticReport search service filters in implementations today. This LOINC code is document-level and misplaced there.
	DiagnosticReport.Code is bound in FHIR core (Valueset-report-codes - FHIR v5.0.0) to Document codes. This LOINC code is bound as a document type (LOINC), so it fits better as a slice in this field.




Impact:

	Semantic drift: Category becomes overloaded with a document code, hampering existing simple queries (category = IMG) relied upon by existing FHIR queries
	Conflicts with core FHIR/LOINC guidance above, as well as other guidance and implementations:
	
		IHE IMR - IMR DiagnosticReport - Interactive Multimedia Report (IMR) v1.1.0
		MIO in Germany - DiagnosticReport - KBV-Basis-Profile - MIO
		IPS - DiagnosticReport (IPS) - International Patient Summary Implementation Guide v2.0.0
	
	




Proposal: Move this LOINC to Code

- category&#91;imaging&#93;.coding&#91;0&#93; = { system: http://loinc.org, code: 18748-4 }

+ code.coding&#91;?system=&#39;http://loinc.org&#39;&#93; = 18748-4  // Diagnostic imaging study
+ category&#91;imaging&#93;.coding&#91;0&#93; = { system: http://terminology.hl7.org/CodeSystem/v2-0074, code: IMG, display: "Imaging" }

 
Proposal 2:

Additionally, we suggest expanding Category&#91;imaging&#93;.Code:

Many groups may have ‘imaging’ to share which they would not normally categorize as ‘diagnostic imaging’. This could include cardiology exams, interventional radiology, etc.


Expand category&#91;imaging&#93; from a single fixed code to the full imaging subset of v2-0074 DiagnosticServiceSectionCodes
Proposal of these code:
+      IMG  – Diagnostic imaging (generic)
+      RAD  – Radiology
+      RX   – Radiograph (plain film)
+      CT   – CAT scan
+      RUS  – Radiology ultrasound
+      NMS  – Nuclear medicine scan
+      CUS  – Cardiac ultrasound
+      CTH  – Cardiac catheterization
(All from CodeSystem `http://terminology.hl7.org/CodeSystem/v2-0074`.)  

This lets cardiology, interventional radiology, nuclear med, etc. slot
under the same slice without inventing new rules.
