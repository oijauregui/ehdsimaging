# FHIR-51334: Are Full Fully Compliant WADO-RS endpoints Envisioned (including support for Study/Series Rendered resources), or only Instance level resources?

## Description

mimeType within the ImWadoEndpoint profile suggests full WADO-RS compliant endpoints, which carries additional obligations:

	can support Study/Series rendered resources, in addition to access to DICOM objects.
	/metadata
	/bulkdata
	/thumbnail



The immediate use cases require only access to the DICOM objects, but other EHDS use cases could benefit from these features:

	Patient applications (EHDS patient access of data)
	Non-dicom apps (Simple GP browser system, EHR imaging overview) could benefit from thumbnails or rendered images for reference quality (not diagnostic quality)




Likewise, WADO-RS requires frame level retrieval/rendering, but we don’t see that there are current use-cases for it in the present IG.


We would suggest that this be clarified so that the community can focus on the subset of features that are most important for the immediate workflow goals. Focusing only on Instance resources is in line with work the Multi-Country Working Group on Imaging Information Sharing is currently considering for XC-WADO (XC-WADO - Google Drive)
