# FHIR-56754: DocumentReference-KeyImage and ImagingSelection-KeyImage are parallel; HL7IDR Finding extension targets four types. Pick one.

## Description

Two parallel Key Image profiles (DocumentReference-KeyImageEuImaging and ImagingSelection-KeyImageEuImaging). HL7IDR Finding extension targets four types (Observation, ImagingSelection, DocumentReference, Media). No guidance on which to use. Consumers must implement all.

Proposed resolution:

Pick one Key Image profile (recommend DocumentReference for R4). Narrow Finding extension to Reference(Observation), with image refs via Observation.derivedFrom.

Reporter(s): Dan Schon, Epic
