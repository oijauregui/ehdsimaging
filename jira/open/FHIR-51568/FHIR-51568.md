# FHIR-51568: Composition.BasedOn is Duplicative to DiagnisticReport.BasedOn (Imaging)

## Description

The order (serviceRequest) that produced the imaging result is already linked in DiagnosticReport.BasedOn - adding an extension for Composition.BasedOn is duplicative. 

Since it could only be an identifier, it does not add value to the human-readable composition resource. Unless there is a good reason for it to be in composition that I'm missing - It should be removed.
