# FHIR-56726: §10.2.1.2.2 implies a single accession per DR; clarify that one DR may serve multiple orders

## Description

The expected number of orders in this section is not clear. A DiagnosticReport may serve as a result for multiple distinct ServiceRequests. The text here uses the plural for orders (good) but then indicates a single AccessionNumber. Consider as an example a single result created for a mammogram and a breast ultrasound — two ServiceRequests, one DiagnosticReport.

Proposed resolution:

Clarify that a DiagnosticReport may correspond to multiple ServiceRequests. Perhaps add "For each order the DiagnosticReport contains...".
